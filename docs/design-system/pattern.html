An editorial system for on-demand objects.

This document captures the visual and interaction patterns used across MAAKM.shop. Use it as a reference when implementing new pages, building components, or extending the catalog. Every pattern here is already in production code.

01Principles

Five rules that hold the system together. When a decision feels unclear, return to these.

01

Quiet by default, loud on accent.

The page rests in warm neutrals. Color is reserved for the one thing that matters most on the screen — a CTA, a price, a serif highlight. Never two accents in the same viewport.

02

Photography over illustration.

Real product photography, never SVG mockups in production. Placeholders only exist to hold layout while imagery is being shot.

03

Editorial spacing.

Generous vertical rhythm (56–80px between sections). Headlines have room to breathe. Tight density only inside cards and tables.

04

Mixed type voice.

Sans-serif for body and UI, italic serif for emphasis inside headlines. The serif is a punctuation mark — one or two phrases per page, never a full headline.

05

Honest about production.

The shop is print-on-demand. Reflect that everywhere: lead times on cards, "no waste" notes at checkout, real material names instead of marketing labels.

12Imagery

Real product photography on neutral backgrounds. Warm window light, slight shadow, never floating-on-white.

DoPhotograph products on warm neutral surfaces (linen, plaster, sand). Use natural light. Show one item per shot in cards; lifestyle context in hero.
Don'tUse AI-generated imagery, e-commerce floating-on-white, or 3D renders pretending to be photos.

14Tokens — drop-in CSS

Copy this into a tokens.css in your frontend. Everything in the system resolves through these custom properties.

/* MAAKM design tokens · v1.0 */
:root {
  /* surface */
  --bg: #F6F4EF;
  --bg-2: #EFEBE2;
  --paper: #FFFFFF;

  /* ink */
  --ink: #1A1A17;
  --ink-2: #4A4A45;
  --ink-3: #8A8A82;

  /* lines */
  --line: #D9D4C7;
  --line-2: #EBE6DB;

  /* accent (themeable) */
  --accent: #D65A31;
  --accent-2: #2A3F2E;

  /* type */
  --font-sans: "Inter Tight", system-ui, sans-serif;
  --font-serif: "Instrument Serif", serif;
  --font-mono: "JetBrains Mono", ui-monospace, monospace;

  /* spacing */
  --s-2xs: 4px;  --s-xs: 8px;
  --s-sm:  12px; --s-md: 16px;
  --s-lg:  24px; --s-xl: 32px;
  --s-2xl: 56px; --s-3xl: 80px;

  /* radius */
  --r-xs: 4px;  --r-sm: 6px;
  --r-md: 8px;  --r-lg: 12px;
  --r-pill: 999px;

  /* motion */
  --ease: cubic-bezier(.2,.7,.2,1);
  --d-fast: 150ms;
  --d-base: 250ms;
  --d-slow: 300ms;
}

/* one global rule the rest of the app inherits from */
body {
  background: var(--bg);
  color: var(--ink);
  font-family: var(--font-sans);
}

Implementation notes

  • Framework agnostic. Tokens are plain CSS variables — works in React, Vue, Svelte, plain HTML. Tailwind users: paste into theme.extend or use @theme in v4.
  • Theme switching works by overriding --accent and --accent-2 on the html element.
  • Fonts load from Google Fonts: Inter+Tight:wght@400;500;600&family=Instrument+Serif:ital@0;1&family=JetBrains+Mono:wght@400;500.
  • Iconography uses Lucide-style line icons at 1.5px stroke. Keep stroke weight consistent across all icons in a single viewport.
